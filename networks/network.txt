## Многоуровненвый подход 
 - задача разбивается на подуровни 
 -  связь с помощью межсетевого интерфейса(интерфейс услуг - определяет набор функций,передаваемый от нижнего к высшему уровню)
---
## Модель OSI 
 - 7 уровней
```
Прикладной (AH) ... (HTTP/SMTP/HTTPS/FTP) (сообщения )
Представление (PH) ... (SSL сертификат)
Сеансовый (SH) ... (SSH) 
Транспортный (TH) ... (датаграммы/пакет) (port/port mapping) (TCP/UDP)
Сетевой (NH) ... (датаграммы) (IP/ARP/ICMP/DHCP)
Канальный (DH) ... (фреймы) (MAC/Ethernet/Wi-Fi)
Физический ... (DT) (биты)

```
 - Прикладной (Взаимодействие сети и пользователя )
 - Представление (преобразование,кодирование/декодирование)
 - Сеансовый (Поддердка длительного взаимодействия (сеанса) ,синхронизация передачи данных)
 - Транспортный (Установка надежного соединения ,контроль ошибок , контроль потока данных )(принять данные,разбить при необходимости и передать нижнему уровню)
 - Сетевой (Объединение сетей в составную сеть (Маршрутизация, Адресация ))
 - Канальный (Управление доступом к среде,исправление ошибок в кадрах)
 - Физический (Преобразование входящих сигналов в биты)

## Модель TCP/IP
 - 4 уровня (получена путем обьединения уровней модели OSI)
 - популярный стек сетевых протоколов 
 - метод получения доступа к сети Internet
 - Основа для создания Intranet - корпоративной сети
 - все современные OC поддерживаю данный стек
 - гибкая технология для сервисов и систем(клиент - сервер и т.д.)
```
Уровень приложений (поток \/ )
Транспортный (поток /\ )
Уровень сети Интернет(межсетевой)
Уровень доступа к сети

```

# Маршрутизация во 2 лабе 		
- Каждый интерфейс маршрутизатора обладает MAC адресом
- IP адреса при передаче из одной подсети в другую могут не меняться ,в отличие от MAC адресов 
- Маршрутизатор - это аппаратное или аппаратно-программное  устройство для обьединения сегментов сети в сеть
- Маршрутизация - процесс выбора пути/путей передачи пакетов между двумя конечными узлами составной сети
- OSPG,BGP
